# ARCH-008: Engineering Standards & Development Playbook

## Purpose

Define engineering standards for implementation, testing, documentation,
and maintenance.

## Core Principles

-   Business value first
-   Simplicity
-   Clear ownership
-   Explicit contracts
-   Fail clearly
-   Build for maintainability

## Approved Toolchain

  Tool         Purpose
  ------------ --------------------------
  Black        Formatting
  Ruff         Linting
  Pyright      Static type checking
  pytest       Testing
  pre-commit   Local quality automation

## Definition of Done

-   Architecture respected
-   Tests pass
-   Formatting and linting pass
-   Documentation updated
-   Meaningful commit created

## Interview Talking Point

Engineering quality was standardized before implementation through
documented practices, automated tooling, and consistent review criteria.
